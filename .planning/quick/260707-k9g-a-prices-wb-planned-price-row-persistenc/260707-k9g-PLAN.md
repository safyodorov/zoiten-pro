Три вещи:
1. Новые поля `WbCard.plannedSellerPrice / plannedSellerDiscountPct` + ручная миграция (локального Postgres нет — применяется на проде через deploy.sh).
2. Отдельная ценовая строка **«Плановая»** в `/prices/wb` — сразу под «Текущая», по умолчанию = текущей, жёлто-оранжевая плашка + бейдж, редактируемая через модалку, персистится на WbCard.
3. Интеграция плановой цены как **базы цены** в план продаж (`lib/sales-plan/data.ts`) — ВЫСОКИЙ РИСК (влияет на живые планы), осознанно смещает базу с funnel-avg на текущую/плановую цену.

Purpose: плановые цены становятся базой, на которой строятся планы продаж (спека §1).
Output: миграция + planned-строка + savePlannedPrice + sales-plan base-price shift, задеплоено с применением миграции.

**КРИТИЧЕСКАЯ семантика (locked для этого плана):**
`plannedSellerPrice` хранит **ФИНАЛЬНУЮ цену продавца** (после скидки продавца) — те же единицы, что `WbCard.price` и `CalculatedPrice.sellerPrice`. Это нужно для консистентности агрегата плана продаж `plannedSellerPrice ?? card.price` (обе — финальные). Модалка вводит «Цену продавца» (финальную), `priceBeforeDiscount` восстанавливается через `deriveBefore`. Не путать с `sellerPriceBeforeDiscount` (Цена для установки) в UI-строке — она восстанавливается из финальной.

**2a. Новый server action `savePlannedPrice`** в `app/actions/pricing.ts` (по паттерну updateProductDelivery — inline-валидация, без новой Zod-схемы):
```ts
/** Фаза A: сохранить плановую цену продавца (ФИНАЛЬНАЯ, после скидки) + скидку на WbCard.
 *  null/null → сброс (плановая = текущей). */
export async function savePlannedPrice(
  wbCardId: string,
  sellerPrice: number | null,
  sellerDiscountPct: number | null,
): Promise<ActionResult> {
  try {
    await requireSection("PRICES", "MANAGE")
  } catch (e) {
    const authErr = handleAuthError(e)
    if (authErr) return authErr
    return { ok: false, error: (e as Error).message }
  }
  if (!wbCardId || wbCardId.length === 0) return { ok: false, error: "wbCardId обязателен" }
  if (sellerPrice != null && sellerPrice < 0) return { ok: false, error: "Цена не может быть отрицательной" }
  if (sellerDiscountPct != null && (sellerDiscountPct < 0 || sellerDiscountPct > 100))
    return { ok: false, error: "Скидка должна быть в диапазоне [0, 100]" }
  try {
    await prisma.wbCard.update({
      where: { id: wbCardId },
      data: {
        plannedSellerPrice: sellerPrice,
        plannedSellerDiscountPct: sellerDiscountPct != null ? Math.round(sellerDiscountPct) : null,
      },
    })
    revalidatePath("/prices/wb")
    return { ok: true }
  } catch (e) {
    return { ok: false, error: (e as Error).message }
  }
}
```
