# Quick 260702-j52: Производство в /stock автоматически из закупок — Summary

**One-liner:** ProductIncoming.orderedQty теперь machine-managed — денормализуется из открытых закупок (PLANNED+ACTIVE) по формуле `Σ max(0, quantity − WAREHOUSE.qty)` при каждой мутации закупки; ручной ввод количества закрыт в /stock и /purchase-plan, число в /stock получило tooltip-раскладку по закупкам со ссылкой на /procurement/purchases.

## Что сделано

### Task 1 — lib/production-sync.ts + скрипт + тесты (1ed109d)
- `computeProductionTotals(items)` — чистый агрегатор: `Σ max(0, quantity − warehouseQty)` per productId, отрицательные позиции клампятся в 0.
- `recomputeProductionForProducts(db, productIds)` — читает открытые позиции (PLANNED+ACTIVE), seed нулями для товаров, выпавших из всех открытых закупок, upsert ТОЛЬКО `orderedQty` (expectedDate/plannedSalesPerDay сохраняются).
- `recomputeAllProduction(db)` — union всех ProductIncoming + всех открытых позиций (обнуляет устаревшие ручные значения).
- PrismaClient через DI (`import type` only) — tsx-скрипт не резолвит alias `@/*`.
- `scripts/recompute-production.ts` — idempotent, логирует до/после/изменено, прод-команда в шапке.
- `tests/production-sync.test.ts` — 6 тестов (simple/partial/clamp/sum/multi/empty), зелёные.

### Task 2 — recompute из server actions закупок (5dc8473)
- `createPurchase` — recompute по productId новых items после транзакции.
- `updatePurchase` — union старых (до транзакции) + новых productId: покрывает удалённые позиции и смену статуса ACTIVE→COMPLETED.
- `deletePurchase` — productId читаются ДО delete (каскад стирает items).
- `savePurchaseItemStages` — select расширен до `{id, productId}`, recompute после перезаписи этапов → частичная приёмка (WAREHOUSE) немедленно уменьшает Производство.
- Каждый вызов в собственном try/catch: `console.error("[production-sync] recompute failed:", e)`, action остаётся ok:true.
- `revalidateProductionLinked()` — /stock + /purchase-plan + /sales-plan после каждой мутации.
- `Product.ivanovoStock` не затронут — закрытие закупки НЕ добавляет остаток к Иваново.

Ширина ячейки w-[88px] и text-right tabular-nums сохранены. Дата прихода и Иваново — редактируемые как раньше.
