<objective>
Сделать «Производство» в /stock производным от закупок: `ProductIncoming.orderedQty` становится machine-managed и пересчитывается из открытых закупок (PLANNED + ACTIVE) по формуле `Σ max(0, PurchaseItem.quantity − qty этапа WAREHOUSE)`. Ручной ввод количества Производства закрывается (input → read-only число + tooltip с раскладкой). Дата прихода на склад Иваново и `Product.ivanovoStock` НЕ трогаются — закрытие закупки не добавляет остаток к Иваново.

Purpose: убрать двойной ручной ввод — количество в производстве теперь всегда актуально и согласовано с реальными закупками и их этапами; частичная приёмка сразу отражается.
Output: денормализующий модуль `lib/production-sync.ts`, миграционный скрипт, обновлённые server actions закупок, обновлённые /stock data + UI, тесты.
</objective>

```ts
export function computeProductionTotals(
  items: Array<{ productId: string; quantity: number; warehouseQty: number }>
): Map<string, number> {
  const totals = new Map<string, number>()
  for (const it of items) {
    const remaining = Math.max(0, it.quantity - it.warehouseQty)
    totals.set(it.productId, (totals.get(it.productId) ?? 0) + remaining)
  }
  return totals
}
```
