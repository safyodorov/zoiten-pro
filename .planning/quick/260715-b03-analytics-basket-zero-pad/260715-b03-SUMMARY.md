# Summary — Analytics card-scan zero-pad basket host

## Что сделано

Прод-баг: прогон ниши FAILED, 4 SKU «нет фото, нет характеристик». Все 4 — однозначные basket (1-9):
код строил `basket-3.wbbasket.ru` (без zero-pad) — такого хоста нет в DNS (http=000) → card.json
недоступен. Двузначные basket (≥10) работали, поэтому падали только low-vol товары.

Диагноз подтверждён на VPS: `basket-3` → http=000, `basket-03` → http=200. Node fetch к CDN
работает (не TLS/UA-блок); парсер card.json корректен (проверено на рабочем SKU 899301731,
media.photo_count=18 / options.len=24). Эталон — bozon `get_basket` (`str(n).zfill(2)`).

Правки (в одном файле `lib/analytics/wb-card-scan.ts` + его тест):
1. **`basketHostName(shard)`** — `basket-${padStart(2,"0")}.wbbasket.ru`. Экспорт.
2. **`cardJsonUrl`** fallback-host → `basketHostName(basketHostForVol(vol))`.
3. **`hostFromMainPhoto`** — regex `^(?:https?:)?\/\/([^/]+)` (ловит `//host`).
4. **`hostCandidates`** — `shardOf(mainPhoto-host) ?? карта` → соседи `[p, p+1..+4, p-1, p-2]` → все через `basketHostName`. Даёт padding И SSRF-нейтрализацию (чужой домен → shardOf=null → карта, host всегда basket-NN.wbbasket.ru).
5. **`CARD_FETCH_HEADERS`** (браузерный UA + Accept) в fetch.
