# Quick 260715-b03 — Analytics card-scan: zero-pad basket host (basket-03, не basket-3)

## Проблема (найдена на проде)

Прогон ниши `cmrlpk8d0biujvh4796hgcotq` → **FAILED**: «Сбор топ-10 неполный: nmId 35783233
(нет фото, нет характеристик)». `incompleteSkus` = 4 SKU, ВСЕ — однозначные basket (1-9):

| nmId | vol | shard | zoiten строил | реально WB |
|---|---|---|---|---|
| 9836784 | 98 | 1 | `basket-1` | `basket-01` |
| 49923608 | 499 | 4 | `basket-4` | `basket-04` |
| 60584140 | 605 | 4 | `basket-4` | `basket-04` |
| 35783233 | 357 | 3 | `basket-3` | `basket-03` |

Проба на VPS: `basket-3.wbbasket.ru` → **http=000** (DNS не резолвится), `basket-03.wbbasket.ru` → **http=200**.
Node fetch к CDN работает (не TLS/UA-блок). card.json парсится корректно (media.photo_count + options —
проверено на рабочем SKU 899301731). Ломалось ТОЛЬКО построение host для shard 1-9 (без zero-pad).
Один упавший SKU в топ-10 по выручке → правило полноты → весь прогон FAILED.

Эталон рабочей механики — bozon `src/analysis/wb/helpers.py::get_basket` (возвращает `str(n).zfill(2)`).

## Задача 1 — zero-pad + протокол-относительный mainPhoto + widen probe + UA

**Файл:** `lib/analytics/wb-card-scan.ts`

- `basketHostName(shard)` → `basket-${padStart(2,"0")}.wbbasket.ru` (новый экспорт).
- `cardJsonUrl` fallback-host: `basketHostName(basketHostForVol(vol))`.
- `hostFromMainPhoto`: regex `^(?:https?:)?\/\/([^/]+)` — ловит и `//host`.
- `hostCandidates`: `shardOf(hostFromMainPhoto(...) ?? "") ?? basketHostForVol(vol)` → соседи `[p, p+1..+4, p-1, p-2]` → все через `basketHostName`.
- `CARD_FETCH_HEADERS` (браузерный UA + Accept) в fetch card.json.

**Verify:** tsc; `basketHostName(3)==="basket-03.wbbasket.ru"`; `cardJsonUrl(35783233)` содержит `basket-03`.
**Done:** shard 1-9 даёт padded host; SSRF-safe (host всегда basket-NN.wbbasket.ru).
