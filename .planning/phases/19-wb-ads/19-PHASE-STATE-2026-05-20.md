## 🎯 Variant A research result (2026-05-20)

**Root cause найден:** `STATS_RELEVANT_STATUSES = {4, 7}` интерпретировались как «running+paused», но по WB official docs (paste пользователя 2026-05-20 от dev.wildberries.ru/docs/openapi/promotion):

| status | Real meaning | (was assumed) |
|---|---|---|
| 4 | **Готова к запуску** (Ready) | "Running" — НЕВЕРНО |
| 7 | **Завершена** (Completed) | "Paused" — НЕВЕРНО |
| 9 | **АКТИВНА** (Running) | "Completed" — НЕВЕРНО |
| 11 | **НА ПАУЗЕ** (Paused) | "Draft/Ready" — НЕВЕРНО |
| -1 | Удалена | (correct) |
| 8 | Отменена | (correct) |

Прежний sync запрашивал /fullstats для Ready(1) + Completed(195) = 196 кампаний → 0 текущего spend. Active (73) + Paused (160) = **233 кампании никогда не sync'ались**.

**Гипотеза «236k₽ через /adv/v2/auto/* (auto-РК)» — ОПРОВЕРГНУТА:**
- WB docs: type 8 (auto-РК) **отключён 23 октября 2025**
- Все auto-РК теперь type 9 с полем `bid_type=unified` (видно через `/api/advert/v2/adverts`)
- Они уже были в наших 388 type=9 — просто status=9, который мы не sync'али

**Verification (sanity check 2026-05-20 17:05 МСК):**
- Backfill 1 batch (50 paused кампаний, days=3) → `statsUpserted: 20, targetsUpserted: 6, total spend 1765.70₽`
- Top campaign 28475171 (paused) spent 1753₽ across 3 nm × 3 days
- До fix'a — `statsUpserted: 0` для тех же 50 ids
