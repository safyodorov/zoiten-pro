SP-16 — Явное предупреждение о срезе плана в матрице «Товары».

Движок УЖЕ режет план по стоку (orders = min(ставка, сток)) и считает per-день `rateRequested` (ставка ДО сток-лимита) и `ordersUnits` (после лимита), плюс `firstStockoutDate` / `lostUnitsToStockout` / `lostRubToStockout` per товар (engine.ts ~272-283). Но в матрице «Товары» ячейка месяца показывает срезанное «П ₽» без причины. Добавляем presentational-объяснение: бейдж «срезано −X% · приход dd.mm» на ячейке месяца, где план ИМЕННО ЭТОГО месяца урезан выше порога; полностью нулевой месяц из-за отсутствия товара → плашка «нет товара · приход dd.mm». Данные уже в движке — расчёт НЕ меняем (D-3).

**Per-month точность (правка checker WARNING):** доля потерь считается ПО МЕСЯЦУ из `planResult.days` (Σ(rateRequested − ordersUnits) по дням месяца × цена), а НЕ по всему горизонту товара. Иначе бейдж «срезано» ложно появлялся бы на месяцах ПОСЛЕ прихода, восстановившего сток (у которых собственного недолива нет). `planResult.days` уже сериализуется в `tableProducts[].planResult` целиком (page.tsx ~239-246 мапит `pr` полностью, `days: PlanDayRow[]` внутри) — новых полей в payload добавлять не требуется, движок не трогается.

Purpose: пользователь сразу видит ПОЧЕМУ конкретный месяц показывает меньше запланированного (сток кончился в этом месяце / приход поздно) и НА СКОЛЬКО (−X%), а не идёт в модалку «Дни».
Output: подтверждение наличия planResult.days на клиенте, per-month расчёт среза из дней месяца, вычисление «ближайшего прихода» из arrivals, бейдж «срезано −X% · приход dd.mm» / плашка «нет товара» в non-edit ячейке месяца ProductPlanTable. Никаких Prisma/движок изменений.

```typescript
/** Первый приход строго после reference-даты (стокаут или today). ISO или null. */
function nextArrivalAfter(arrivals: ArrivalBatch[], ref: string): string | null {
  const future = arrivals.map((a) => a.date).filter((d) => d > ref).sort()
  return future[0] ?? null
}

/** "2026-09-28" -> "28.09" */
function fmtDayMonth(iso: string): string {
  const [, m, d] = iso.split("-")
  return `${d}.${m}`
}

const MONTH_SHORT: Record<string, string> = {
  "01": "янв", "02": "фев", "03": "мар", "04": "апр", "05": "май", "06": "июн",
  "07": "июл", "08": "авг", "09": "сен", "10": "окт", "11": "ноя", "12": "дек",
}
/** "2026-09-01" -> "сен" */
function fmtMonthShort(iso: string): string {
  return MONTH_SHORT[iso.slice(5, 7)] ?? iso.slice(0, 7)
}

/**
 * Per-month срез из дней движка. Возвращает недолив ЭТОГО месяца в единицах и ₽,
 * и его долю от планового спроса месяца (rateRequested). Данные из planResult.days.
 * planUnits — Σ ordersUnits (после сток-лимита), requested — Σ rateRequested (до лимита).
 */
function monthShortfall(
  days: PlanDayRow[],
  monthPrefix: string,     // "2026-09"
  avgPriceRub: number,
): { lostUnits: number; lostRub: number; lostShare: number } {
  let requested = 0, filled = 0
  for (const d of days) {
    if (!d.date.startsWith(monthPrefix)) continue
    requested += d.rateRequested
    filled += d.ordersUnits
  }
  const lostUnits = Math.max(0, requested - filled)
  return {
    lostUnits,
    lostRub: lostUnits * avgPriceRub,
    lostShare: requested > 0 ? lostUnits / requested : 0,
  }
}
```
