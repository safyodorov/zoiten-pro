# Phase 21: Кредиты — Context

**Gathered:** 2026-06-08
**Status:** Ready for planning

<domain>
## Phase Boundary

Phase 21 добавляет раздел `/credits` — учёт и визуализация всех кредитов компании. Новая БД кредитов с графиками погашения; UI из трёх частей: список кредитов → детальная карточка кредита → сводный горизонтальный график выплат с настраиваемой разбивкой (день / неделя / месяц).

**В scope:**
- Новые модели `Loan` + `LoanPayment` (+ справочник `Bank`)
- Новый `ERP_SECTION.CREDITS` (в enum его сейчас нет) + RBAC
- Раздел `/credits` — список кредитов с ключевыми числами/датами + CRUD через модалку
- `/credits/[id]` — детальная карточка: сводные числа + таблица графика + line-chart остатка
- Сводный горизонтальный график выплат (`/credits/schedule` или таб) — разбивка день/неделя/месяц, группировка по организации с подытогами + Итого
- Разовый seed-скрипт для импорта текущих данных из `Кредиты.xlsx`

**Не в scope (deferred / future):**
- Постоянная UI-кнопка импорта Excel (нестабильный формат файла) — только разовый seed
- Автогенерация графика по аннуитету (храним явные строки)
- Уведомления о датах платежей (cron email/Telegram)
- Раздел «Балансы» и «Новые кредиты» из Excel как отдельные таблицы (на текущем этапе — только база кредитов + графики)
- Экспорт / печать графика

</domain>

### Загрузка данных (D-01)

- **D-01 (обновлён U-01/U-04):** Основной механизм — **ручной ввод через CRUD-форму** (кредиты добавляются редко). Текущие кредиты заливаются **разовым seed-скриптом** (`scripts/seed-credits.ts`). Источник: **детальные графики из папки `Кредиты/`** (Сбербанк XLSX через `xlsx`-пакет + JetLend PDF через `pdftotext -layout`) для строк `LoanPayment`; метаданные/контрольные суммы — из `Кредиты.xlsx` Лист2. НЕ строим устойчивый парсер в UI — постоянная кнопка импорта deferred.

### Модель данных — график погашения (D-02..D-04)

- **D-02:** График хранится **явными строками** `LoanPayment` — одна строка на каждую дату платежа. Платежи в Excel нерегулярные (не чистый аннуитет), поэтому формульный расчёт не подходит.
- **D-03:** Гранулярность — **по фактическим датам платежей** (`date` + `principal` + `interest`), как в Excel Лист1. Бакетирование день/неделя/месяц для сводного графика считается **на лету** при рендере, а не хранится.
- **D-04:** Остаток основного долга и сводные числа (погашено тела, уплачено процентов, текущий остаток) **вычисляются** накопительно из `Loan.amount` и строк `LoanPayment`. Тело и проценты **задаются явно** (из Excel / формы), система их не пересчитывает по ставке. → исключает рассогласование с банком, но даёт корректные агрегаты.

### Модель данных — кредит (D-05..D-08)

- **D-05:** Модель `Loan`:
  - `id String @id @default(cuid())`
  - `contractNumber String` (№ КД, напр. «№ 3702242101-23-2» или «№21220» — free text, форматы разные)
  - `companyId String` FK → `Company` (организация: Пеликан/Зойтен/Сикрет Вэй/Дрим Лайн → существующие записи Company; `onDelete: Restrict` или `SetNull` — решит planner)
  - `lenderId String` FK → `Lender` (новый справочник кредиторов; **переименован из `bankId`/`Bank`** — см. U-03)
  - `amount Decimal @db.Decimal(14, 2)` (сумма кредита, тело)
  - `annualRatePct Decimal @db.Decimal(6, 3)` (годовая ставка %, напр. 28.000; в Excel хранится как 0.28 → при seed умножаем на 100)
  - `termMonths Int?` (срок в месяцах, напр. 24; nullable — есть не у всех)
  - `issueDate DateTime?` (дата выдачи — **ручное поле, nullable**; см. D-07)
  - `notes String?` (заметки, опционально)
  - `createdAt / updatedAt / deletedAt` (soft delete как у Product/Supplier)
- **D-06:** Модель `LoanPayment`:
  - `id String @id @default(cuid())`
  - `loanId String` FK → `Loan` (`onDelete: Cascade`)
  - `date DateTime` (дата платежа)
  - `principal Decimal @db.Decimal(14, 2) @default(0)` (тело долга)
  - `interest Decimal @db.Decimal(14, 2) @default(0)` (проценты)
  - `@@index([loanId, date])`
  - Остаток в строке НЕ хранится — вычисляется при рендере.
- **D-07:** **Дата выдачи** — отдельное ручное поле `issueDate` (nullable). Если не заполнено → UI показывает дату первого платежа из графика как ориентир (fallback при отображении, не пишется в БД). При seed из Excel оставляем null (графики стартуют ~апрель 2024, точная дата выдачи неизвестна).
- **D-08 (обновлён U-03):** Новый справочник `Lender { id, name @unique, sortOrder Int @default(0), createdAt, updatedAt }` («Кредитор» — банки + краудлендинг + др.). Значения из данных: **Сбербанк** (договоры 3702…, источник `Кредиты.xlsx` + Сбербанк XLSX) и **JetLend** (числовые договоры 20xxx–29xxx, источник PDF `support@jetlend.ru`). Возможны другие. Управляется как Brand/Category (DnD reorder опционально, минимум — CRUD).

### Статус кредита (D-09)

- **D-09:** Статус **вычисляется автоматически** по остатку: `активен` если остаток тела > 0, `погашён` если остаток = 0 (или все плановые платежи в прошлом и остаток ≤ 0). Без ручного ведения статуса. Хранится как computed (не поле БД) либо cached — решит planner.

### Routes & RBAC (D-10, D-11)

- **D-10:** Новый `ERP_SECTION.CREDITS` (добавить в enum — **сейчас отсутствует**). Routes:
  - `/credits` → список кредитов (RSC) + CRUD модалка
  - `/credits/[id]` → детальная карточка кредита
  - Сводный график — таб/подраздел на `/credits` (напр. переключатель «Список | Сводный график») или `/credits/schedule`. Точную форму (таб vs отдельный route) решит planner; CLAUDE.md tabs-паттерн (CardsTabs / AdsTabs).
- **D-11:** RBAC:
  - Read (списки, карточка, сводный график): `requireSection("CREDITS")`
  - Write (все server actions, seed-триггер если через UI): `requireSection("CREDITS", "MANAGE")`

### Список кредитов (D-12)

- **D-12:** Таблица кредитов с ключевыми колонками: организация, кредитор, № КД, сумма, ставка %, срок, дата выдачи, текущий остаток, статус. (Фильтры: организация, кредитор, статус.) Sticky-таблица (raw HTML, не shadcn `<Table>` — CLAUDE.md). Клик по строке → `/credits/[id]`. Каскадные/простые фильтры: организация, банк, статус (planner детализирует).

### Сводный горизонтальный график выплат (D-13..D-17) — ЦЕНТРАЛЬНАЯ ФИШКА

- **D-13:** Горизонтальная таблица: крайний левый блок (sticky, не скроллится) = информация по кредиту, далее вправо — колонки-периоды с выплатами в ячейках.
- **D-14:** Разбивка периодов — **настраиваемая: день / неделя / месяц** (переключатель). Бакетирование считается на лету из `LoanPayment.date`.
- **D-15:** **Каждый кредит = две строки**: одна для **тела долга**, одна для **процентов**. В ячейках периодов — соответствующие суммы (сумма principal / сумма interest, попавших в бакет периода).
- **D-16:** Группировка строк — **по организации** (Пеликан / Зойтен / Сикрет Вэй / Дрим Лайн) с **подытоговыми строками по каждой орг** (тело + проценты) + общая строка **Итого** внизу (как Excel Лист2 per-org + Итого).
- **D-17:** Левый sticky-блок содержит: **организация + кредитор + № КД + сумма + ставка % + текущий остаток**. Клик по строке кредита → детальная карточка.
- **D-13a:** Диапазон периодов — **настраиваемый диапазон дат** (поля «с / по») + выбранная разбивка. По умолчанию — разумное окно (напр. текущий месяц ± несколько периодов; planner определит дефолт). Горизонтальный скролл по колонкам-периодам.

### Детальная карточка кредита (D-18)

- **D-18:** `/credits/[id]` показывает:
  1. **Сводные карточки** (summary cards, паттерн `components/ads/SpendSummary.tsx`): сумма кредита, погашено тела, уплачено процентов, текущий остаток, переплата (сумма процентов), ставка/срок/даты.
  2. **Таблица графика**: дата / тело / проценты / (вычисленный) остаток основного долга по каждой строке.
  3. **Line-chart остатка** основного долга во времени (recharts, как `WbAdvertOrdersChart` в Phase 19).

### Numeric & locale (D-19)

- **D-19:** Деньги — `Decimal @db.Decimal(14, 2)`; ставка — `Decimal @db.Decimal(6, 3)` (поддержка 28.000). Отображение ru-RU, тысячи с пробелом, «₽» справа (паттерн `/ads/wb`, `/procurement`). Даты — МСК через существующий helper (`getMskTodayString` / `lib/wb-cron-schedule.ts`).

### Claude's Discretion

- Точная навигация сводного графика (таб на `/credits` vs отдельный route `/credits/schedule`).
- Дефолтный диапазон/окно сводного графика и дефолтная разбивка (предложение: месяц).
- Расположение пункта в Sidebar (`nav-items.ts`) + иконка Lucide (предложение: `Landmark` / `Banknote` / `CreditCard`).
- Дефолтная сортировка списка (предложение: организация → банк → дата выдачи).
- Цветовое кодирование статуса (активен — синий, погашён — emerald/серый).
- `Bank` — нужен ли DnD reorder в `/admin/settings` или достаточно простого CRUD.
- `onDelete` поведение FK `Loan.companyId` / `Loan.bankId` (Restrict vs SetNull).
