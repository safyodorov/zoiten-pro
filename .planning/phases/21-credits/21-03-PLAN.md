    1. Типы:
    ```typescript
    export type LoanGranularity = "day" | "week" | "month"
    export interface PaymentInput { date: Date | string; principal: number; interest: number }
    export interface ScheduleRow { date: Date; principal: number; interest: number; balance: number }
    export interface LoanAggregates {
      totalPrincipalPaid: number
      totalInterestPaid: number
      currentBalance: number   // amount − Σprincipal

    }
    export type LoanStatus = "active" | "paid"
    ```

    2. `export function computeSchedule(amount: number, payments: PaymentInput[]): ScheduleRow[]`
       - сортировать payments по date ASC
       - накопительно: `cumulativePrincipal += p.principal; balance = round2(amount − cumulativePrincipal)`
       - вернуть строки в порядке дат

    3. `export function computeLoanAggregates(amount: number, payments: PaymentInput[]): LoanAggregates`
       - `totalPrincipalPaid = round2(Σ principal)`
       - `totalInterestPaid = round2(Σ interest)`
       - `currentBalance = round2(amount − totalPrincipalPaid)`
       - `overpayment = totalInterestPaid`

    4. `export function computeStatus(currentBalance: number): LoanStatus`
       - `return currentBalance > 0 ? "active" : "paid"` (D-09)

    5. `export function bucketKey(date: Date, granularity: LoanGranularity): string`
       - "day" → `date.toISOString().slice(0,10)` (YYYY-MM-DD)
       - "month" → YYYY-MM
       - "week" → ISO 8601 week key `YYYY-Www` (понедельник = начало недели). Реализовать через стандартный ISO week algorithm.

    6. `export function bucketLabel(key: string, granularity: LoanGranularity): string`
       - человекочитаемая ru-RU метка колонки (день: «09.06», неделя: «нед. 24», месяц: «июн 2026»). Для месяца — короткие RU названия месяцев.

    7. helper `round2(n: number): number` = `Math.round(n * 100) / 100`.

    Не использовать `Date` в SSR-небезопасный способ — работать с UTC-датами (даты платежей хранятся как @db.Date, без времени). Для бакетирования использовать UTC компоненты.
