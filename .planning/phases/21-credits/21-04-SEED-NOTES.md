## 2. Структура Сбербанк XLSX

**Файлы:** 2 шт.
- `График_платежей_по_договору_3702242101_23_2_на_08_06_2026_г_.xlsx`
- `График_платежей_по_договору_3702259264_23_2_на_08_06_2026_г_.xlsx`

**Лист:** `График платежей` (единственный)

**Структура строк:**
- Row 0: `"График платежей по договору № {contractNumber}"` — источник № КД
- Row 1: `" на 08.06.2026 г."` — дата выгрузки (только хвост)
- Row 2: пустая
- Row 3 (заголовки): `Дата | К оплате, ₽ | Основной долг, ₽ | Проценты, ₽ | Остаток основного долга, ₽`
- Rows 4+: данные платежей

**Порядок колонок (индексы 0-4):**
- 0: Дата (через `cellDates: true` → JS Date; или строка `"DD.MM.YYYY"`)
- 1: К оплате ₽ (общая сумма платежа)
- 2: Основной долг ₽ (principal)
- 3: Проценты ₽ (interest)
- 4: Остаток основного долга ₽ (remaining balance)

**Формат чисел:** `399,557.24` — **ЗАПЯТАЯ = разделитель тысяч, ТОЧКА = десятичная**.
При `xlsx` с `cellDates: true` числа читаются напрямую как JS number (не строка).

**Важно (U-05):** Выгрузка «на 08.06.2026» содержит **только оставшиеся платежи**.
- 3702242101-23-2: rows 4-5 (2 строки: проценты 02.07.2026 + финальный платёж 03.07.2026). Оба остатка = 0.
- 3702259264-23-2: rows 4-5 (2 строки: проценты 18.06.2026 + финальный платёж 19.06.2026). Оба остатка = 0.
Полная история нужна из Лист2.

## 3. Структура JetLend PDF

**Файлы:** 11 PDF в папке `Кредиты/`

**Парсинг:** `pdftotext -layout "<путь>" -` → stdout

**Структура:**
- Строка 1 (или первая непустая): номер договора (короткое число, напр. `25397`)
- Служебный блок: реквизиты ЮЛ (ИНН, счета, support@jetlend.ru и т.д.)
- Заголовок таблицы: строка с `,` маркерами (пустая практически)
- Данные платежей: **чередующиеся строки** — со строкой даты и строкой без даты

**Строка платежа (с датой):**
```
DD.MM.YYYY  total  principal  interest  0,00  0,00  annuity  fee  remaining_balance
```
Regex: `/^(\d{2}\.\d{2}\.\d{4})\s+([\d\s]+,\d{2})\s+([\d\s]+,\d{2})\s+([\d\s]+,\d{2})/`

Колонки по позиции (после даты):
1. Общая сумма к оплате
2. **Основной долг** (principal) — брать
3. **Проценты** (interest) — брать
4. 0,00 (штрафы/пени)
5. 0,00 (другое)
6. Аннуитетный платёж
7. Комиссия
8. **Остаток основного долга** (remaining) — для сверки

**Формат чисел (русский):** `504 678,64` — **ПРОБЕЛ (и неразрывный пробел U+00A0) = разделитель тысяч, ЗАПЯТАЯ = десятичная**.
Нормализация: убрать все `\s` + ` `, заменить `,` на `.`, затем `parseFloat`.

**Строки-промежутки (без даты):** содержат sub-period-level данные (не нужны для seed — берём только строки с датой).

**Источник № договора:** первая непустая строка в stdout pdftotext (после trim). Числовой формат (20967, 25397 и т.д.).

## 8. Решение U-05: дополнение истории Сбербанка

Детальные Сбербанк XLSX содержат только хвост (2 строки каждый — платежи за июнь-июль 2026).
История с начала кредита берётся из Лист2 помесячно.

**Для seed:**
1. Парсим Лист2 помесячные данные для Сбер-контрактов (Зойтен col58, ДрЛайн col62, СикрВэй col66, Пеликан col70 + соседние interest cols 59,63,67,71)
2. Из детального XLSX берём только те строки, которых НЕТ в Лист2 (т.е. платёж ПОСЛЕ декабря 2026)
3. Сверка: накопленный Σprincipal из Лист2 + XLSX = amount на момент текущей выгрузки

**Дата строки (решение для помесячных записей):** используем **первое число месяца** (`new Date(Date.UTC(year, month-1, 1))`). Это стандартный способ представить «период» без привязки к конкретному дню, и не конфликтует с детальными PDF-строками (которые на конкретные даты).
