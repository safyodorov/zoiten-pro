**scripts/systemd/zoiten-erp.service** — main app service (per D-15 through D-19):
```
[Unit]
Description=Zoiten ERP (Next.js)
After=network.target postgresql.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/zoiten-pro
EnvironmentFile=/etc/zoiten.pro.env
Environment=NODE_ENV=production
Environment=PORT=3000
ExecStart=/usr/local/bin/node /opt/zoiten-pro/.next/standalone/server.js
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

Key: `EnvironmentFile=/etc/zoiten.pro.env` is where DATABASE_URL, AUTH_SECRET, AUTH_URL, CRON_SECRET, UPLOAD_DIR live. NODE_ENV and PORT are set inline (not secret). PATH must include `/usr/local/bin` for node.

**scripts/systemd/zoiten-purge.service** — one-shot cron (per D-31, D-32):
```
[Unit]
Description=Zoiten ERP — purge soft-deleted products (30-day cleanup)
After=network.target zoiten-erp.service

[Service]
Type=oneshot
EnvironmentFile=/etc/zoiten.pro.env
ExecStart=/usr/bin/curl -s -o /dev/null -w "%%{http_code}" \
  -H "x-cron-secret: ${CRON_SECRET}" \
  http://127.0.0.1:3000/api/cron/purge-deleted
```

Note the `%%{http_code}` — in systemd unit files `%` must be doubled to escape it.

**scripts/systemd/zoiten-purge.timer** — daily at 02:00:
```
[Unit]
Description=Run Zoiten purge daily at 02:00

[Timer]
OnCalendar=*-*-* 02:00:00
Persistent=true

[Install]
WantedBy=timers.target
```

**deploy.sh** — idempotent deploy script (per D-13, D-14, D-12):
```bash
#!/usr/bin/env bash
set -euo pipefail

APP_DIR="/opt/zoiten-pro"

echo "==> Pulling latest code..."
cd "$APP_DIR"
git pull

echo "==> Installing dependencies..."
npm ci --omit=dev

echo "==> Running database migrations..."
npx prisma migrate deploy

echo "==> Building application..."
npm run build

echo "==> Copying static assets to standalone..."
cp -r public .next/standalone/public
cp -r .next/static .next/standalone/.next/static

echo "==> Restarting service..."
systemctl restart zoiten-erp

echo "==> Done. Checking status..."
systemctl status zoiten-erp --no-pager -l
```

The `cp -r public` and `cp -r .next/static` steps are required for standalone output — Next.js does NOT auto-copy these on build. Without them, CSS/JS and public assets 404 in production.

Make deploy.sh executable: the file should be committed without execute bit — the developer runs `chmod +x deploy.sh` on the VPS (or runs `bash deploy.sh`).

**scripts/nginx/zoiten-pro.conf** (per D-20 through D-25, Pitfall 8 client_max_body_size):

```nginx
# Zoiten ERP — nginx reverse proxy config
# Place at: /etc/nginx/sites-available/zoiten-pro
# Enable:   ln -s /etc/nginx/sites-available/zoiten-pro /etc/nginx/sites-enabled/zoiten-pro
# Validate: nginx -t
# Reload:   systemctl reload nginx
#
# IMPORTANT: Run `nginx -T` before enabling to understand existing CantonFairBot config.
# Never edit /etc/nginx/nginx.conf directly — use sites-available only.

server {
    listen 80;
    server_name zoiten.pro www.zoiten.pro 85.198.97.89;

    client_max_body_size 5m;

    # Product photos — served directly by nginx, never hits Node.js
    # Matches the UPLOAD_DIR=/var/www/zoiten-uploads/ env var
    location /uploads/ {
        alias /var/www/zoiten-uploads/;
        expires 30d;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    # All other requests → Next.js on port 3000
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}

# SSL — uncomment after `certbot --nginx -d zoiten.pro -d www.zoiten.pro` runs
# certbot will auto-modify this file to add HTTPS redirect and SSL block
# Prerequisite: zoiten.pro DNS must point to 85.198.97.89 before running certbot
```
