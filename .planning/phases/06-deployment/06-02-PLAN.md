The document must cover these sections in order:

**1. Prerequisites**
- VPS: root@85.198.97.89 (key-based SSH — no password prompt expected)
- Local: git repo cloned, SSH key configured
- Reminder: CantonFairBot is at /opt/CantonFairBot/ — do NOT break it

**2. First-time VPS setup**
Sub-sections:
- 2.1 Check existing nginx config (CRITICAL — do this first)
  - `ssh root@85.198.97.89 "nginx -T 2>&1 | head -60"`
  - Note what domains/ports are currently configured
  - Confirm CantonFairBot config file location
- 2.2 Install Node.js 22.x LTS (per D-03)
  - Check if already installed: `node --version`
  - If not: install via NodeSource: `curl -fsSL https://deb.nodesource.com/setup_22.x | bash - && apt-get install -y nodejs`
  - Verify: `node --version` (expect v22.x.x), `npm --version`
- 2.3 Install PostgreSQL 16 (per D-04)
  - `apt install -y postgresql postgresql-contrib`
  - Start + enable: `systemctl enable --now postgresql`
- 2.4 Create PostgreSQL database and user (per D-05, D-06)
  - Switch to postgres user and run psql commands:
    ```
    sudo -u postgres psql <<'SQL'
    CREATE USER zoiten WITH PASSWORD 'CHANGE_ME_SECURE_PASSWORD';
    CREATE DATABASE zoiten_erp OWNER zoiten;
    GRANT ALL PRIVILEGES ON DATABASE zoiten_erp TO zoiten;
    SQL
    ```
  - Note: use the same password in /etc/zoiten.pro.env DATABASE_URL
- 2.5 Create upload directory (per D-29)
  - `mkdir -p /var/www/zoiten-uploads`
  - `chown -R www-data:www-data /var/www/zoiten-uploads` (or `root:root` if running as root)
  - `chmod 755 /var/www/zoiten-uploads`
- 2.6 Clone application (per D-08)
  - `git clone https://github.com/safyodorov/zoiten-pro /opt/zoiten-pro`
  - `cd /opt/zoiten-pro`

**3. Configure environment variables (per D-09, Pitfall 14)**
- Create /etc/zoiten.pro.env (NOT inside the project directory):
  ```
  cp /opt/zoiten-pro/.env.example /etc/zoiten.pro.env
  nano /etc/zoiten.pro.env
  ```
- What to fill in:
  - DATABASE_URL: use the password from step 2.4
  - AUTH_SECRET: `openssl rand -hex 32`
  - AUTH_URL: `http://85.198.97.89:3000` (change to https://zoiten.pro after domain pointed)
  - CRON_SECRET: `openssl rand -hex 32`
  - UPLOAD_DIR: `/var/www/zoiten-uploads`
- Security note: `chmod 600 /etc/zoiten.pro.env` — secrets should not be world-readable

**4. Install dependencies, run migrations, seed, build (per D-12)**
```bash
cd /opt/zoiten-pro
npm ci --omit=dev
npx prisma migrate deploy    # NOT migrate dev — never in production
npx prisma db seed           # Idempotent — safe to run again
npm run build
# Copy static assets (required for standalone output):
cp -r public .next/standalone/public
cp -r .next/static .next/standalone/.next/static
```

**5. Install systemd service and timer (per D-15 through D-19, D-31, D-32)**
```bash
cp /opt/zoiten-pro/scripts/systemd/zoiten-erp.service /etc/systemd/system/
cp /opt/zoiten-pro/scripts/systemd/zoiten-purge.service /etc/systemd/system/
cp /opt/zoiten-pro/scripts/systemd/zoiten-purge.timer /etc/systemd/system/
systemctl daemon-reload
systemctl enable --now zoiten-erp
systemctl enable --now zoiten-purge.timer
```
- Verify service is running: `systemctl status zoiten-erp`
- Verify timer is active: `systemctl list-timers zoiten-purge.timer`
- Tail logs: `journalctl -u zoiten-erp -f`

**6. Configure nginx (per D-20 through D-24, Pitfall 13)**
- CRITICAL: Verify CantonFairBot nginx config location (from step 2.1) before proceeding
```bash
cp /opt/zoiten-pro/scripts/nginx/zoiten-pro.conf /etc/nginx/sites-available/zoiten-pro
ln -s /etc/nginx/sites-available/zoiten-pro /etc/nginx/sites-enabled/zoiten-pro
# Validate BEFORE reloading:
nginx -t
# If nginx -t passes:
systemctl reload nginx
```
- Verify Zoiten is accessible: `curl -I http://85.198.97.89` (expect 200 or redirect)
- CRITICAL: Verify CantonFairBot still works after nginx reload

**9. Re-deploy (day-2 operations)**
After initial setup, future deploys run:
```bash
ssh root@85.198.97.89 "cd /opt/zoiten-pro && bash deploy.sh"
```
The deploy.sh script: git pull → npm ci → prisma migrate deploy → npm run build → copy standalone assets → systemctl restart zoiten-erp

**10. Troubleshooting**
- App not starting: `journalctl -u zoiten-erp -n 50 --no-pager`

- Login fails: check AUTH_SECRET and AUTH_URL in /etc/zoiten.pro.env
- DB connection error: verify DATABASE_URL password matches PostgreSQL user
