# Plan 17-02 — Settings UI

## Goal

Суперадмин в `/admin/settings → Категории` может для каждой категории определить набор свойств (имя, тип, опции, mapping на WB), drag-and-drop порядок, удаление с cascade-предупреждением. В `/admin/settings → Направления` — чекбокс `hasSizes`.

## Tasks

### T1: Server actions (extend app/actions/reference.ts)

5 новых actions:

- `createCategoryProperty({ categoryId, name, kind, options[], wbAttrName? })` → CreateResult
- `updateCategoryProperty({ id, name, kind, options[], wbAttrName? })` → ActionResult
- `deleteCategoryProperty(id)` → ActionResult (cascade удалит ProductPropertyValue)
- `reorderCategoryProperties(ids[])` → ActionResult
- `setDirectionHasSizes({ directionId, hasSizes })` → ActionResult

Все требуют `requireSuperadmin()`. Все revalidatePath `/admin/settings` + `/products` (форма товара зависит от свойств).

Schema validation через Zod:
- `name`: 1..100 chars
- `kind`: enum STRING/ENUM/NUMBER
- `options`: max 50 items, каждый 1..100 chars; required если kind=ENUM
- `wbAttrName`: max 100 chars, optional

### T2: DirectionsTab — добавить hasSizes toggle

**File:** `components/settings/DirectionsTab.tsx`

В `DirectionRow` (рядом с именем направления) добавить чекбокс «Имеет размеры»:

```tsx
<label className="flex items-center gap-1.5 text-xs">
  <Checkbox
    checked={direction.hasSizes}
    onCheckedChange={(c) => handleToggleSizes(!!c)}
    disabled={isPending}
  />
  Имеет размеры
</label>
```

Props `DirectionWithBrands` extend: `hasSizes: boolean`. Server action: `setDirectionHasSizes`.

### T3: CategoriesTab — добавить inline свойства

**File:** `components/settings/CategoriesTab.tsx`

При раскрытии Brand → Categories — каждая category показывает (помимо подкатегорий) блок «Свойства»:

```
▼ Брюки
   Подкатегории:
   • Длинные
   • Шорты
```

Состояние редактирования — inline form с native `<select>` для kind, textarea для options (comma-separated), input для wbAttrName.

### T4: SettingsTabs props update

`SettingsTabs.tsx` принимает расширенные типы:
- `directions[].hasSizes: boolean`
- `brands[].categories[].properties: CategoryProperty[]`

`/admin/settings/page.tsx` обновить fetch:
- `prisma.brand.findMany({ include: { categories: { include: { subcategories: ..., properties: { orderBy: { sortOrder: "asc" } } } } } })`
- `prisma.productDirection.findMany({ include: { brands: ... } })` уже включает все нужные fields после миграции (`hasSizes` появляется автоматически)

## Out of Scope

- Свойства на «Уровне товара» (т.е. свойство не привязанное к категории) — deferred. Сейчас если у товара нет категории, у него нет и свойств.
- Inheritance свойств от Category → Subcategory — deferred. Сейчас свойства жёстко на Category.
- Импорт/экспорт CategoryProperty между категориями — deferred.
