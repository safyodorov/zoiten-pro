# Plan 17-03 — Product UI

## Goal

В форме `/products/[id]/edit`:
- После Подкатегории — секция «Свойства» с динамическими полями (по CategoryProperty текущей категории)
- После Свойств — секция «Размерная сетка» (видима только если у направления товара `hasSizes=true`)
- Кнопка «Импортировать из WB» — диалог-предпросмотр с разделением «свойства» / «размеры» / «что подтянется» / «что уже есть».

## Tasks

### T1: Server actions (app/actions/products.ts extend)

- `saveProductProperties({ productId, values: [{ propertyId, value }] })` → ActionResult
  - upsert по `@@unique([productId, propertyId])` 
  - value="" → удалить значение
- `saveProductSizes({ productId, sizes: [{ value, sortOrder }] })` → ActionResult
  - clean-replace: deleteMany + createMany в транзакции
- `importFromWb({ productId, replaceExisting: boolean })` → ImportResult
  - Найти основной MarketplaceArticle товара (WB slug, sortOrder=0)
  - По nmId найти WbCard, прочитать `characteristics` + `techSizes`
  - Для каждого CategoryProperty с wbAttrName — найти соответствующее значение в characteristics
  - Если `replaceExisting=false`: не трогать существующие ProductPropertyValue
  - Создать ProductSize для каждого techSize (skip существующие по unique constraint)
  - Вернуть `{ properties: { matched, applied, skipped }, sizes: { added, skipped } }`
- `previewWbImport(productId)` → PreviewResult (read-only — что подтянется без записи)

Все защищены `requireSection("PRODUCTS", "MANAGE")`.

### T2: ProductForm — секция «Свойства»

**File:** `components/products/ProductForm.tsx`

Новые props:
- `categoryProperties: CategoryProperty[]` — все свойства для выбранной категории (передаются с сервера в page)
- `initialPropertyValues: { propertyId, value }[]` — текущие значения

Секция после Подкатегории:

```tsx
{watchedCategoryId && categoryProperties.length > 0 && (
  <section className="space-y-4">
    <h2 className="text-lg font-medium border-b pb-2 flex items-center justify-between">
      Свойства
      <Button onClick={openWbImportDialog}>Импортировать из WB</Button>
    </h2>
    {categoryProperties.map(prop => (
      <PropertyInput key={prop.id} property={prop} form={form} />
    ))}
  </section>
)}
```

Тип `PropertyInput`:
- STRING → `<Input />`
- ENUM → `<NativeSelect>` со списком `options`
- NUMBER → `<Input type="number" />`

Значения хранятся в form state как `properties.{propertyId}: string`.

### T3: ProductForm — секция «Размерная сетка»

Видимость: `currentBrand?.direction?.hasSizes === true`.

Field array через `useFieldArray({ name: "sizes" })`. Каждая строка: input + drag handle + delete.

```tsx
{currentBrand?.direction?.hasSizes && (
  <section className="space-y-4">
    <h2 className="text-lg font-medium border-b pb-2">Размерная сетка</h2>
    {sizeFields.map((f, i) => (
      <SortableSizeRow key={f.id} index={i} form={form} />
    ))}
    <Button onClick={() => appendSize({ value: "", sortOrder: sizeFields.length })}>
      + Добавить размер
    </Button>
  </section>
)}
```

DnD pattern — копия из MarketplaceGroupInline (PointerSensor + dnd-kit), уже знакомый паттерн.

### T4: WB Import Dialog

**File:** `components/products/WbImportDialog.tsx`

Открывается из кнопки в секции «Свойства». Шаги:

1. Mounted → `previewWbImport(productId)` → показывает 4-секцию:
   - Свойства, которые подтянутся (mapping CategoryProperty.wbAttrName → WbCard.characteristics[].value)
   - Свойства БЕЗ wbAttrName или БЕЗ совпадения → «Не найдено в WB»
   - Размеры из WbCard.techSizes — новые (которых нет у товара) + уже существующие
   - Чекбокс «Перезаписать существующие свойства» (default false)
2. Кнопка «Импортировать» → `importFromWb(...)` → toast с итогом «Свойства: 2 применены, 1 пропущено. Размеры: 3 добавлены, 1 уже был.»
3. После success → `router.refresh()` → форма перерисуется с новыми значениями.

### T5: Pages update

**Files:**
- `app/(dashboard)/products/[id]/edit/page.tsx`
- `app/(dashboard)/products/new/page.tsx`

```tsx
// На edit page:
const product = await prisma.product.findUnique({
  where: { id },

// Все категории передаются как раньше, но extend brands include с properties:

  include: {
    direction: true,  // включает hasSizes

        subcategories: ...,

  },

```

ProductForm derive `categoryProperties` из `currentBrand.categories.find(c => c.id === watchedCategoryId).properties`.

После deploy:
1. Создать тестовый товар → бренд Men's Factor → категория Брюки → отображается секция «Свойства» с полями Пол (select) и Цвет (input)
