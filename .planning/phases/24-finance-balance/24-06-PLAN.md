  где marketplace.name.toLowerCase()==="wb" — паттерн lib/stock-data.ts:177-197 (батч
  wbCard.findMany({where:{nmId:{in}}}) + Map<nmId, card>).
- WbCard (schema:326-328): stockQty (WB_WAREHOUSE), inWayToClient (WB_IN_WAY_TO_CLIENT),
  inWayFromClient (WB_IN_WAY_FROM_CLIENT) — суммировать по всем nmId товара.

- ProductCost.costPrice (schema:446-450, Float, 1:1 через product.cost) → costPriceAtDate;
  нет записи → null (D-11), valueRub = null; есть → valueRub = round2(qty × costPrice).
- Products: where deletedAt:null; строки с qty===0 по локации НЕ писать (экономия объёма).

Дата снапшота (D-02 «утром за вчера»): вчерашняя дата МСК как @db.Date:
`const d = new Date(Date.now() + 3*3600_000 - 24*3600_000); const dateStr = d.toISOString().split("T")[0]`
→ `new Date(dateStr)` (полночь UTC — консистентно с @db.Date, паттерн wb-funnel-daily mskDateMinusDays).
⚠ КОНВЕНЦИЯ ДАТЫ (m6 — едина с 24-05/24-09): `date` снапшота = состояние на КОНЕЦ этого дня.
Cron в 06:00 дня D фиксирует конец ВЧЕРА (D−1) и пишет `date = D−1`. loadBalanceSheet(asOf) читает
снапшот `date = asOf` + live `date <= asOf` (обе — конец дня asOf). Дрейфа при сверке нет.

Понедельник недели снапшота МСК (для forPay-хвоста, M2 — якорить на ДАТУ СНАПШОТА, не запуска!):
от `snapshotDate` (вчера МСК, см. выше) отнять (isoDay-1) дней, isoDay = jsDay===0 ? 7 : jsDay
(паттерн lib/loan-math.ts getIsoWeek). Иначе воскресный/понедельничный запуск теряет неделю дебиторки.
Хвост суммирует продажи с saleDt ∈ [monday(snapshotDate), snapshotDate] (пост-фильтр — см. 24-03 M1).

export interface StockSnapshotRowInput {
  productId: string; sku: string; name: string

  qty: number; costPriceAtDate: number | null; valueRub: number | null
}
// PURE (без prisma) — тестируется с фикстурами:
export function computeStockSnapshotRows(

    costPrice: number | null; nmIds: number[] }>,

): StockSnapshotRowInput[]
// Оркестратор (prisma + wb-finance-api):
export async function runFinanceSnapshot(): Promise<{ date: string; stockRows: number; receivables: "ok" | "degraded" | "skipped" | "error" }>
