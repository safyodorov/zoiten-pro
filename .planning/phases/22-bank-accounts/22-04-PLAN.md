Логика `persistParsedTransactions`:
1. **Owning Bank (ДЕТЕРМИНИРОВАНО):** `const ob = OWNING_BANK[opts.sourceBank]`; `const owningBank = await prisma.bank.upsert({ where: { bic: ob.bic }, update: {}, create: { bic: ob.bic, name: ob.name } })`. Один на весь батч. **Это единственный источник `BankAccount.bankId`** — никакого извлечения БИК из шапки, никаких placeholder'ов, никакого guessing.
2. **Counterparty Bank (отдельный путь):** банк контрагента из колонки БИК выписки (`tx.counterpartyBic`) upsert-ится своей записью `Bank` по реальному БИК: для каждого уникального непустого `counterpartyBic` → `prisma.bank.upsert({ where: { bic }, update: {}, create: { bic, name: bic } })` (name = bic, редактируется позже вручную). Кешировать в Map. Эти Bank-записи НЕ используются для `BankAccount.bankId` — только наполняют справочник банков-контрагентов. (Если в схеме 22-01 у `BankTransaction`/`Counterparty` нет FK на банк контрагента — достаточно создать запись Bank в справочнике; денорм. `counterpartyBic` всё равно пишется в `BankTransaction`.)
3. **Company:** для каждой уникальной (companyInn||companyName): если companyInn заполнен → `prisma.company.upsert({ where: { inn: companyInn }, update: {}, create: { name: companyName ?? companyInn, inn: companyInn } })`. Если ИНН нет → найти по name (`findFirst({ where: { name } })`); если нет — создать с name. Кешировать в Map по ключу.
4. **BankAccount:** `upsert({ where: { number: accountNumber }, update: { currency }, create: { number, currency, companyId, bankId: owningBank.id } })`. `bankId` всегда = `owningBank.id` (из шага 1). Кешировать по number.
5. **Counterparty:** если counterpartyInn заполнен → `upsert({ where: { inn }, update: {}, create: { inn, name: counterpartyName ?? inn } })`; иначе counterpartyId = null (денорм. поля counterpartyName/Inn/Bic/Account всё равно пишутся в BankTransaction). Кешировать по inn.
6. Создать `ImportBatch` запись (fileName=opts.fileName, sourceBank=opts.sourceBank, rowsTotal=parsed.length, rowsImported=0, rowsSkipped=0, importedById=opts.importedById ?? null).
7. Подготовить массив для `createMany`: каждой tx → `{ accountId, date, direction, amount, currency, docNumber, operationType, purpose, counterpartyId, counterpartyName, counterpartyInn, counterpartyBic, counterpartyAccount, category: "UNCATEGORIZED", fingerprint: computeFingerprint(tx), importBatchId, sourceBank }`.
   - Дедуп ВНУТРИ батча: `Map<fingerprint, row>` перед записью (skipDuplicates не ловит дубли внутри одного createMany на некоторых версиях).
8. `const result = await prisma.bankTransaction.createMany({ data: rows, skipDuplicates: true })`. `result.count` = реально вставлено.
9. Обновить ImportBatch: `rowsImported = result.count`, `rowsSkipped = parsed.length - result.count`.
10. Вернуть `{ imported: result.count, skipped: parsed.length - result.count, total: parsed.length }`.

Money: `amount` записывать как number — Prisma примет в Decimal поле. date: ParsedTransaction.date — уже Date (UTC midnight), пишется в @db.Date.

```typescript
"use server"
import { requireSection } from "@/lib/rbac"
import { prisma } from "@/lib/prisma"
import { revalidatePath } from "next/cache"
import type { TxCategory } from "@prisma/client"

type ActionResult = { ok: true } | { ok: false; error: string }

function handleAuthError(e: unknown): { ok: false; error: string } | null {
  if (e instanceof Error) {
    if (e.message === "UNAUTHORIZED") return { ok: false, error: "Не авторизован" }
    if (e.message === "FORBIDDEN") return { ok: false, error: "Нет доступа" }
  }
  return null
}

const VALID_CATEGORIES: TxCategory[] = [
  "UNCATEGORIZED","INTERNAL_TRANSFER","BANK_FEE","SUPPLIER_PAYMENT","INCOME","TAX","LOAN","OTHER",
]

export async function categorizeTx(id: string, category: string): Promise<ActionResult> {
  try {
    await requireSection("BANK", "MANAGE")
    if (!VALID_CATEGORIES.includes(category as TxCategory)) {
      return { ok: false, error: "Недопустимая категория" }
    }
    await prisma.bankTransaction.update({ where: { id }, data: { category: category as TxCategory } })
    revalidatePath("/bank")
    return { ok: true }
  } catch (e) {
    const authErr = handleAuthError(e)
    if (authErr) return authErr
    if ((e as { code?: string })?.code === "P2025") return { ok: false, error: "Операция не найдена" }
    console.error("categorizeTx error:", e)
    return { ok: false, error: "Ошибка сервера" }
  }
}
```
