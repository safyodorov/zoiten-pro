## Схема (Prisma) — на финализацию в plan-phase

```prisma
enum CashDirection { INCOME EXPENSE }   // приход / расход

model CashCategory {
  id        String      @id @default(cuid())
  name      String      @unique
  sortOrder Int         @default(0)
  entries   CashEntry[]
  createdAt DateTime    @default(now())
  updatedAt DateTime    @updatedAt
}

model CashEntry {
  id                   String        @id @default(cuid())
  date                 DateTime      @db.Date
  direction            CashDirection
  amount               Decimal       @db.Decimal(14, 2)
  department           String?
  categoryId           String?
  category             CashCategory? @relation(fields: [categoryId], references: [id], onDelete: SetNull)
  purpose              String        @db.Text          // назначение (исходный текст)
  responsibleEmployeeId String?
  responsibleEmployee  Employee?     @relation(fields: [responsibleEmployeeId], references: [id], onDelete: SetNull)
  responsibleNameRaw   String?                         // исходная фамилия из файла
  comment              String?       @db.Text
  source               String        @default("manual") // 'budget-yulya' | 'budget-pavel' | 'manual'
  fingerprint          String?       @unique           // дедуп импорта
  createdAt            DateTime      @default(now())
  updatedAt            DateTime      @updatedAt

  @@index([date])
  @@index([categoryId])
  @@index([responsibleEmployeeId])
  @@index([direction])
}
```
+ `Employee` back-relation `cashEntries CashEntry[]`.
+ `ERP_SECTION.CASH` в enum.

**Дедуп импорта (fingerprint):** SHA-256 от `sheet | date(YYYY-MM-DD) | direction | amount(2dp) | normalize(purpose) | responsibleNameRaw`. Идемпотентный повторный импорт. Ручные записи (`source=manual`) — fingerprint null (не мешают дедупу).
