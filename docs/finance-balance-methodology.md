# Методология баланса (/finance/balance)

Справка по тому, как считается каждая строка управленческого баланса Zoiten ERP.
Эта же справка доступна прямо на странице баланса — кнопка **«Как считается»**
([components/finance/BalanceMethodologyDialog.tsx](../components/finance/BalanceMethodologyDialog.tsx)).
Источник правил — [lib/balance-data.ts](../lib/balance-data.ts) (`loadBalanceSheet`).

Баланс строится на выбранную дату: **Активы = Пассивы + Капитал** (Капитал — балансирующая
строка). Часть статей берётся из ночного снапшота, часть вычисляется на лету на дату.

## ⚠ Дебиторка WB = «Баланс» из кабинета WB (реальное время)

Строка «Дебиторка Wildberries» = поле `current` из **Balance API**
(`finance-api.wildberries.ru/api/v1/account/balance`) — деньги, которые WB уже должна продавцу.

- Обновляется **постоянно, в реальном времени** по мере выкупов (официальная инструкция WB
  «Как работать с балансом»). Совпадает с виджетом «Баланс» в кабинете продавца.
- **Уже включает выкупы текущей (незакрытой) недели.** Поэтому незакрытая неделя отдельной
  строкой НЕ прибавляется — иначе те же выкупы посчитались бы дважды.
- Не путать с «Суммой к выводу» (`for_withdraw`) — это меньшая величина: сколько можно вывести
  прямо сейчас (недельный/3-недельный цикл), а не сколько WB должна всего.
- Заказы «в пути к клиенту» (ещё не выкуплены) в `current` не входят — это будущая дебиторка,
  но они уже учтены в балансе как **запас по себестоимости**, поэтому в дебиторку повторно не
  добавляются.

Подробный разбор (почему `current + weeklyTail` был двойным счётом):
[.planning/debug/wb-receivables-double-count.md](../.planning/debug/wb-receivables-double-count.md).

## Активы

| Статья | Как считается |
|---|---|
| **Банковские счета (₽)** | Остаток рублёвых счетов на дату: закрывающий баланс выписки ± банковские транзакции до выбранной даты. |
| **Банковские счета (CNY)** | Справочно, НЕ входит в рублёвый итог — валютная переоценка не выполняется (v1). |
| **Касса** | Приходы − расходы кассы на выбранную дату. |
| **Дебиторка Wildberries** | = `current` из Balance API WB (см. выше). Из ночного снапшота дебиторки; нет снапшота → «нет снапшота на дату». |
| **Запасы (склады)** | Склады WB / в пути к клиенту / в пути от клиента / Иваново — количество из снапшота остатков × себестоимость на дату. Товары без себестоимости в сумму не входят (плашка «Без оценки»). |
| **Товар в пути из Китая** | Оплаченные закупки на этапе «Отгрузка»/«Транзит» (ещё не приняты на склад). Оплата приводится в ₽ по курсу ЦБ на дату платежа. |
| **Авансы поставщикам** | Оплаченные закупки ДО отгрузки (этап «Производство»/«Инспекция» или без этапа). |
| **Ручные статьи (актив)** | Введённые вручную корректировки, действуют с указанной даты. |

## Пассивы

| Статья | Как считается |
|---|---|
| **Кредиты и займы** | Остаток по кредитам на дату (тело − погашенная часть). Кредит, не выданный на дату, не учитывается. |
| **Отложенные налоги (расчётно)** | Начислено (выкупы × ставки НДС и налога на доходы) − уплачено (платежи с категорией «Налоги» в банке и кассе). Для закрытых кварталов можно ввести факт. Приближённая оценка. |
| **Ручные статьи (пассив)** | Введённые вручную корректировки. |

## Капитал

**Капитал = Активы − Пассивы** — балансирующая строка («чистая стоимость»).

## Оговорки и приближения

- Снапшоты (остатки и дебиторка) пишутся ночным заданием в **06:00 МСК за предыдущий день**;
  остальные статьи считаются на лету на выбранную дату.
- История снапшотов — **с 01.07.2026**; ретроспектива раньше не восстанавливается.
- Курсы валют — ЦБ РФ, накапливаются с 09.06.2026; для более ранних дат — самый ранний
  известный курс (приближение).
- Себестоимость — текущая (истории нет). Кнопка «Пересчитать дату» переоценивает остатки
  снапшота по свежей себестоимости.
- Колонки «на дату» и «сравнение» + дельты Δ₽/Δ% — для отслеживания динамики.
